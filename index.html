Tokens

What is a token?

LLMs don't read letters or words — they read tokens: common chunks of text (a whole word, part of a word, or punctuation). The model learned these chunks from huge amounts of text. Each colored block below is one token. Notice that common words are usually one token, while rare words get split into several.

Token IDs

What is a token ID?

Every token has a number — its ID — its index in the model's vocabulary. The model only ever sees these numbers, not text. Each ID below is colored to match its token above, so you can see which number stands for which chunk.


    


    

      
Context window

      

        What's a context window?
        
A model can only "see" a limited number of tokens at once — its
        context window. Everything (your prompt + its reply) must
        fit. That's why very long documents have to be split up. The bar shows how
        much of a model's window your current text would use.

Tokenizer comparison

      

        Why do token counts differ?
        
Different model families use different tokenizers. Newer ones (GPT-4o)
        pack text into fewer tokens than older ones (GPT-3). And
        because tokenizers learned mostly from English, the same sentence in other
        languages — or emoji — often costs many more tokens. Try
        the examples below and watch the bars change.

      

      

        
        
        
        
        
      

      
      

    


    

      
Vector Graph

      

        What is an embedding / vector?
        
To capture meaning, a model turns each word into a list of
        numbers — a vector — a point in "meaning space". Words
        with similar meanings end up close together. These vectors have hundreds
        of dimensions, so we squeeze them down to 3D (X/Y/Z) to draw them. Words
        that cluster together are ones the model considers related.

Next-token predictor

      

        How does an LLM actually generate text?
        
It's surprisingly simple: an LLM just predicts the next
        token, over and over. Given some text, it outputs a probability
        for every possible next token; it picks one, adds it, and repeats. Below,
        a small real model (distilGPT-2) shows its top guesses for what comes next.
        Click a prediction to add it and see what the model would say next.

Semantic search (mini-RAG)

      

        How does "chat with your documents" work?
        
Each sentence and your question are turned into vectors, then ranked by
        meaning — the closest sentences are the "answer". A chatbot then reads those
        retrieved sentences to reply. This retrieve-then-answer pattern is
        retrieval-augmented generation (RAG), how AI "chats with
        your files".

Image recognition (vision)

      

        How does AI "see"?
        
An image is just a grid of numbers (pixels). A vision model turns those
        numbers into features and predicts labels — the same "turn input into
        numbers, then predict" idea as language models, applied to pictures.

      

      
      
      

      

      

    


    

      
Speech-to-text (Whisper)

      

        How does AI "hear"?
        
Sound is turned into a stream of numbers, and a model (OpenAI's Whisper)
        converts those into text tokens — the same tokens you saw at the top. Click
        record, speak a sentence, and watch it transcribe.
